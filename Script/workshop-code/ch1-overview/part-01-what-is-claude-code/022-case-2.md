# Slide 22: Case 2 리팩토링

**Part 1: WHAT IS CLAUDE CODE**

## Code Blocks

### TYPICAL FLOW

```
User: 모든 axios 호출을 신규 ApiClient 래퍼로 마이그레이션해 주세요.

Claude:
  1. Grep    "axios\.(get|post|put|delete)" → 47 occurrences
  2. Read    각 파일의 호출 패턴 분석
  3. Plan    공통 변환 규칙 도출
  4. Edit    47곳 일괄 변환 (Find&Replace + 문맥 검토)
  5. Bash    npm run lint && npm test
  6. Bash    git diff --stat → 변경 요약 보고
```

## Speaker Notes

두 번째 사례인 리팩토링입니다.
모든 axios 호출을 신규 ApiClient 래퍼로 마이그레이션하는 작업을 가정해봅니다.
Claude Code는 먼저 Grep으로 axios 호출 패턴을 검색해 47개의 변경 대상을 식별합니다.
다음으로 각 파일의 호출 패턴을 Read로 분석하고, 공통 변환 규칙을 Plan 단계에서 정리합니다.
47곳을 일괄로 Edit 도구로 변환하는데, 단순 Find and Replace가 아니라 각 호출의 문맥을 검토하여 안전하게 적용합니다.
마지막으로 lint와 test를 자동 실행하고, git diff로 변경 요약 보고서를 만들어 줍니다.
핵심은 영향 범위 식별, 사전 계획, 일괄 적용, 자동 검증의 4단계 흐름입니다.
이 패턴은 모든 대규모 리팩토링에 동일하게 적용됩니다.
