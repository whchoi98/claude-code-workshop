# Slide 21: Case 1 디버깅

**Part 1: WHAT IS CLAUDE CODE**

## Code Blocks

### TYPICAL FLOW

```
User: 다음 에러가 발생합니다. 원인을 찾고 수정해 주세요.

  TypeError: Cannot read property 'id' of undefined
    at UserService.findById (src/services/user.js:42)
    at AuthController.login  (src/controllers/auth.js:78)

Claude:
  1. Read   src/services/user.js (line 30-60)
  2. Read   src/controllers/auth.js (line 60-90)
  3. Grep   "findById" → 3 callsites
  4. Edit   src/services/user.js  (null check 추가)
  5. Bash   npm test -- user.test.js
```

## Speaker Notes

디버깅 사례를 구체적인 흐름으로 살펴봅니다.
사용자가 TypeError 같은 에러 메시지를 그대로 붙여넣으면, Claude Code는 다음과 같이 자율 작업을 수행합니다.
1단계로 스택 트레이스에 나온 파일을 Read 도구로 읽고, 2단계로 호출자 측 컨트롤러도 함께 읽습니다.
3단계로 findById 함수가 호출되는 모든 지점을 Grep으로 찾아 영향 범위를 파악합니다.
4단계로 null 체크 같은 안전한 수정안을 Edit으로 적용하고, 5단계로 관련 테스트를 즉시 실행해 회귀가 없는지 확인합니다.
핵심은 사용자가 단계마다 지시할 필요 없이 에러 메시지 하나만 제공하면 Claude Code가 알아서 추적과 수정과 검증을 한 번에 처리한다는 점입니다.
