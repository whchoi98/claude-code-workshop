# Slide 23: Case 3 신기능 개발

**Part 1: WHAT IS CLAUDE CODE**

## Code Blocks

### TYPICAL FLOW

```
User: /users API에 페이지네이션을 추가하고 테스트도 작성해 주세요.

Claude:
  1. Read    src/routes/users.js, src/services/user.js, tests/
  2. Plan    page/limit 쿼리, links 헤더, 기본 20, 최대 100
            → 사용자에게 계획 제시
  3. Edit    routes, service, repository 3개 파일 수정
  4. Write   tests/users.pagination.test.js 신규 작성
  5. Bash    npm test → green
  6. Bash    git checkout -b feat/users-pagination
  7. Bash    git commit / gh pr create
```

## Speaker Notes

세 번째 사례는 신기능 개발입니다.
사용자 API에 페이지네이션을 추가하는 작업을 예로 들어봅니다.
Claude Code는 먼저 관련 파일들을 Read로 읽어 현재 구조를 파악하고, Plan 단계에서 페이지네이션 설계, 즉 page와 limit 쿼리 파라미터, links 응답 헤더, 기본 20개, 최대 100개 같은 결정사항을 사용자에게 제시하여 합의를 얻습니다.
합의가 이루어지면 routes, service, repository 세 파일을 한 번에 수정하고, 신규 테스트 파일을 작성합니다.
테스트가 통과하면 새 브랜치를 만들어 커밋하고, gh CLI를 통해 PR까지 자동으로 생성합니다.
즉, 요구사항 한 줄을 받아 검토 가능한 PR까지 일괄로 만들어 주는 흐름입니다.
사용자는 PR을 검토하고 머지만 하면 됩니다.
