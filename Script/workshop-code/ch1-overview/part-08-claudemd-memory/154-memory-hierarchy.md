# Slide 154: Memory Hierarchy

**Part 8: CLAUDE.md & MEMORY**

## Code Blocks

### MEMORY LAYERS

```
Priority Order (높을수록 우선):

  ┌────────────────────────────────────────────────────────┐
  │ 1. Enterprise   /etc/claude/CLAUDE.md           (조직)  │
  │    조직 차원의 정책, 보안 요건, 표준 도구              │
  ├────────────────────────────────────────────────────────┤
  │ 2. User         ~/.claude/CLAUDE.md             (개인)  │
  │    개인 선호, 코드 스타일, 자주 쓰는 명령              │
  ├────────────────────────────────────────────────────────┤
  │ 3. Project      ./CLAUDE.md                  (프로젝트) │
  │    프로젝트 구조, 빌드/테스트 명령, 컨벤션             │
  ├────────────────────────────────────────────────────────┤
  │ 4. Local        ./.claude/local.md      (개인 X 프로젝트)│
  │    .gitignore 처리, 로컬 개발자 비밀 설정              │
  └────────────────────────────────────────────────────────┘

병합: 4계층이 합쳐져 단일 컨텍스트로 Claude에 전달됨
```

## Speaker Notes

Claude Code의 메모리는 4개 계층으로 관리됩니다.
첫째 Enterprise는 /etc/claude/CLAUDE.md에 위치하며 조직 차원의 정책, 보안 요건, 표준 도구를 정의합니다.
둘째 User는 사용자 홈 아래 ~/.claude/CLAUDE.md로 개인 선호, 코드 스타일, 자주 쓰는 명령 등을 담습니다.
셋째 Project는 프로젝트 루트의 ./CLAUDE.md로 프로젝트 구조, 빌드와 테스트 명령, 코드 컨벤션을 정의합니다.
넷째 Local은 .claude/local.md에 위치하며 보통 .gitignore에 포함되어 로컬 개발자만의 비밀 설정을 담습니다.
이 4계층은 자동으로 병합되어 매 세션 시작 시 단일 컨텍스트로 Claude에 전달됩니다.
