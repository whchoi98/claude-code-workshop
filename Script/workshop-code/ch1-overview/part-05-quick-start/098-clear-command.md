# Slide 98: /clear 명령

**Part 5: QUICK START**

## Code Blocks

### /clear

```bash
# 디버깅 작업 완료 후 새로운 기능 개발 시작
> 디버깅이 완료되었습니다. 테스트가 모두 통과합니다.
> /clear
> Context cleared. CLAUDE.md reloaded.

# 새 작업 시작
> 이제 user/profile 페이지에 아바타 업로드 기능을 추가하고 싶어.
[Claude는 이전 작업 맥락 없이 깨끗한 상태로 시작]

# 비교 - /clear 없이 계속하면
> 디버깅이 완료되었습니다.
> 이제 아바타 업로드 기능을 추가해 줘.
[Claude는 디버깅 컨텍스트가 남아 혼란스러울 수 있음]
```

## Speaker Notes

/clear 명령은 작업 전환 시 필수입니다.
한 작업이 끝나고 전혀 다른 주제로 이동할 때 컨텍스트를 완전히 비워 이전 작업의 잔재가 새 작업에 영향을 주지 않게 합니다.
디버깅을 마치고 새로운 기능 개발을 시작할 때 /clear를 사용하지 않으면 Claude가 이전 디버깅 컨텍스트를 일부 활용하려 들어 혼란스러운 결과가 나올 수 있습니다.
/clear 후에는 CLAUDE.md가 자동으로 다시 로드되므로 프로젝트 컨벤션은 그대로 유지됩니다.
부수 효과로 토큰 사용량도 크게 줄어들어 비용 절감 효과도 있습니다.
