# Slide 189: FAQ

**Part 10: RECAP & LABS**

## Code Blocks

### FAQ

```
Q1. Claude Code와 Claude.ai 챗은 무엇이 다른가요?
A.  Claude.ai는 일반 챗 인터페이스, Claude Code는 코딩 전용 에이전트.
    도구 호출, 파일 시스템 접근, 권한 모델 등 코딩 워크플로 최적화.

Q2. 사내망에서 사용하려면 어떻게 해야 하나요?
A.  Bedrock 또는 Vertex AI 경로 + 사내 프록시 설정이 일반적.
    상세는 Chapter 3 Admin Setup에서 다룸.

Q3. 비용이 부담스럽습니다. 어떻게 절감하나요?
A.  (1) Sonnet 기본 사용 (2) /cost 모니터링 (3) 작업 범위 축소
    (4) /clear 적절 활용 (5) Subagent로 작은 모델 위임

Q4. 코드가 학습에 사용되나요?
A.  기본적으로 학습에 사용되지 않음. ZDR 옵션도 제공.

Q5. 다른 LLM도 사용 가능한가요?
A.  Claude Code는 Claude 모델 전용. 단, Bedrock/Vertex 경로로
    동일 모델을 다른 청구 경로로 호출 가능.
```

## Speaker Notes

Chapter 1과 관련해 자주 받는 질문 다섯 가지를 정리했습니다.
첫째 Claude Code와 Claude.ai 챗의 차이는 챗은 일반용, Code는 코딩 전용으로 도구 호출과 파일 시스템 접근, 권한 모델 등 코딩 워크플로에 최적화되어 있다는 점입니다.
둘째 사내망 사용은 Bedrock 또는 Vertex 경로와 사내 프록시 설정이 일반적이며 Chapter 3에서 상세히 다룹니다.
셋째 비용 절감은 Sonnet 기본 사용, /cost 모니터링, 작업 범위 축소, /clear 활용, Subagent 위임의 5가지 방법이 있습니다.
넷째 코드가 학습에 사용되는지에 대한 답은 기본적으로 사용되지 않으며 ZDR 옵션도 있다는 것입니다.
다섯째 다른 LLM 사용은 불가능하나 Bedrock과 Vertex 경로로 동일 모델을 다른 청구로 사용할 수는 있습니다.
더 많은 질문은 공식 문서와 사내 Slack 채널을 활용하시기 바랍니다.
