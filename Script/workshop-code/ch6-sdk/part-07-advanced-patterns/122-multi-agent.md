## Code Blocks

### patterns

```bash
# 왜 멀티 에이전트?
# 1. 컨텍스트 분리 - 각 에이전트는 자기 작업에만 집중
# 2. 전문성 - system prompt를 역할별로 최적화
# 3. 병렬 처리 - 여러 작업을 동시 진행

# 흐름 예시
# user request
#   -> Orchestrator (계획)
#     -> Worker 1 (코드 작성)
#     -> Worker 2 (테스트 작성)
#     -> Worker 3 (문서 작성)
#   -> Critic (검토)
#   -> Final response
```

## Speaker Notes

멀티 에이전트 개요입니다.
3가지 주요 패턴이 있습니다.
Orchestrator-Worker는 상위 에이전트가 작업을 분할하고 worker 에이전트들에 위임한 후 결과를 종합합니다.
Specialist 분리는 각자 전문 분야가 있는 에이전트들입니다.
코드 리뷰, 보안, 성능 같은 분야별 전문가로 나눕니다.
Critic 패턴은 한 에이전트가 작업하고 다른 에이전트가 검토합니다.
품질 향상에 효과적입니다.
왜 멀티 에이전트를 사용하는지 4가지 이유가 있습니다.
컨텍스트 분리로 각 에이전트는 자기 작업에만 집중합니다.
전문성으로 system prompt를 역할별로 최적화할 수 있습니다.
병렬 처리로 여러 작업을 동시 진행합니다.
검증으로 결과를 다른 에이전트가 교차 검증합니다.
흐름 예시를 보면 user request가 들어오면 Orchestrator가 계획을 세우고 Worker 3명이 코드, 테스트, 문서를 병렬로 작성한 후 Critic이 검토해 최종 응답을 만듭니다.
