# Slide 137: FAQ

**Part 9: RECAP & 4 LABS**

## Code Blocks

### FAQ

```
Q1. settings.json을 잘못 설정해서 망가지면?
A.  파일을 삭제하면 기본값으로 복원됩니다.
    rm ~/.claude/settings.json 후 claude doctor로 확인.
    managed-settings는 관리자에게 요청.

Q2. Hooks가 너무 느려서 작업이 답답한데?
A.  /tmp/hook-perf.log에 실행 시간을 기록해 측정.
    무거운 작업은 백그라운드(&)로 비동기 분리.
    timeout 조정으로 임시 완화 가능.

Q3. MCP 서버를 npm 미러에 두려면?
A.  npm pack으로 tarball 생성 후 사내 미러에 publish.
    settings.json에서 사내 registry로 자동 사용됨.

Q4. 같은 이름의 명령이 여러 곳에 있으면?
A.  우선순위: 프로젝트 > 사용자 > 빌트인.
    더 상위가 적용되며 /commands로 확인 가능.

Q5. Subagent와 Slash Command 중 선택은?
A.  격리된 큰 분석은 Subagent.
    빠른 워크플로 자동화는 Command.
    한 Command 안에서 Subagent 호출도 가능.

Q6. Hook을 한 환경에만 적용하려면?
A.  Hook 스크립트 안에서 환경변수나 hostname으로 분기.
    또는 .claude/settings.local.json에 등록 (Git 미커밋).
```
