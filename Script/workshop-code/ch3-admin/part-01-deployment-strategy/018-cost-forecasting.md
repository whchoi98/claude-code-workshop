# Slide 18: Cost forecasting

**Part 1: DEPLOYMENT STRATEGY**

## Code Blocks

### Forecast

```bash
# 1. 개발자 1명당 월 평균 (실측 기준)
- 평일 사용: 6시간 × 20일 = 120시간/월
- 시간당 평균 토큰: input 50K, output 5K
- 월 토큰: input 6M, output 0.6M
- Sonnet 단가: $3/M input + $15/M output
- 1인당 월 비용: 6 × $3 + 0.6 × $15 = $27

# 2. 조직 규모별 예상 월 비용
- 10명 팀:     $270/월
- 50명 팀:     $1,350/월
- 200명 조직:  $5,400/월

# 3. 비용 증감 요인
+ Subagent 다용:     +30-50% (병렬 호출)
+ Opus 사용:         +200-400% (해당 작업)
- Haiku 적극 활용:   -40-60%
- Plan 모드 사용:    -10-20% (의도 정확화)

# 4. 안전 마진 권장
실제 도입 시 예상치 + 30% 버퍼 확보
```

## Speaker Notes

도입 전 사용량과 비용을 추정하는 방법입니다.
개발자 1명당 월 평균을 실측 기준으로 계산합니다.
평일 6시간씩 20일을 사용하면 월 120시간입니다.
시간당 평균 토큰은 input 50K, output 5K 정도입니다.
월 토큰은 input 6M, output 0.6M으로 계산됩니다.
Sonnet 단가 기준으로 1인당 월 27달러 정도 나옵니다.
조직 규모별 예상 월 비용은 10명 팀 270달러, 50명 팀 1350달러, 200명 조직 5400달러입니다.
비용 증감 요인이 있습니다.
Subagent를 많이 쓰면 30에서 50퍼센트 증가합니다.
Opus를 사용하면 200에서 400퍼센트 증가합니다.
반면 Haiku를 적극 활용하면 40에서 60퍼센트 절감됩니다.
Plan 모드를 사용하면 10에서 20퍼센트 절감됩니다.
실제 도입 시 예상치에 30퍼센트 안전 마진을 더하는 것을 권장합니다.
