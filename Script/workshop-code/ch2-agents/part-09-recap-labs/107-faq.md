# Slide 107: FAQ

**Part 9: RECAP & LABS**

## Code Blocks

### FAQ

```
Q1. Subagent를 너무 많이 만들면 어떤 문제가 있나요?
A.  메인이 description 매칭으로 후보를 추리므로 너무 많으면 매칭이
    헷갈립니다. 5~10개 이내가 적정. description을 명확히 작성하세요.

Q2. Subagent가 또 다른 Subagent를 호출할 수 있나요?
A.  기술적으로 가능하지만 권장하지 않습니다. 복잡도가 빠르게 증가하고
    디버깅이 어려워집니다. 메인에서 직렬 호출이 더 명확합니다.

Q3. 팀 전체에 Subagent를 어떻게 배포하나요?
A.  .claude/agents를 Git에 커밋하면 됩니다. PR 리뷰로 품질 관리도 가능.

Q4. Subagent가 실수로 위험한 변경을 했을 때 복구는?
A.  Git이 가장 강력한 안전망. 항상 브랜치에서 작업하고 PR로 검토.

Q5. Subagent 비용이 너무 큽니다. 어떻게 줄이나요?
A.  (1) 단순 작업은 Haiku (2) /cost로 모니터링 (3) tools 좁히기
    (4) Opus는 정말 필요할 때만 (5) 결과 캐싱 패턴 도입
```
