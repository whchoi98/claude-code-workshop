## Code Blocks

### Lab 4 workflow

```bash
# 시나리오: PR 자동 검증 워크플로
# .github/workflows/pr-check.yml

name: PR Check with Subagents
on: pull_request

jobs:
  comprehensive-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @anthropic-ai/claude-code

      # 직렬: code-reviewer 먼저 → 결과 보고
      - name: Code Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "@code-reviewer git diff main..HEAD" \
            --output-format text > review.md

      # 병렬: 테스트와 보안 동시
      - name: Tests + Security
        run: |
          claude -p "다음을 모두 점검해 줘: 1) 테스트 커버리지 부족
                     2) 보안 이슈" --output-format json > result.json

      # 결과를 PR 코멘트로 게시
      - uses: peter-evans/create-pr-comment@v1
        with: { body-path: review.md }
```

## Speaker Notes

Lab 4는 멀티 에이전트 협업 실습입니다.
약 30분 소요됩니다.
PR 자동 검증 워크플로를 GitHub Actions로 구현합니다.
구조는 직렬과 병렬의 복합 패턴입니다.
먼저 code-reviewer를 직렬로 호출해 결과를 review.md에 저장합니다.
다음 테스트 커버리지와 보안을 병렬로 점검합니다.
메인이 두 작업을 동시에 처리합니다.
결과는 JSON으로 받아 후처리합니다.
마지막으로 review 결과를 PR 코멘트로 자동 게시합니다.
peter-evans의 create-pr-comment 액션을 활용합니다.
PR 작성자는 자동 코드 리뷰, 테스트 점검, 보안 검토 결과를 한 번에 받을 수 있습니다.
이 패턴은 실무에서 바로 적용 가능합니다.
본인 프로젝트의 워크플로에 적용해 보시기 바랍니다.
